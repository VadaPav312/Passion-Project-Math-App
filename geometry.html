N-RN.1

Rational Exponents

Explain how the definition of the meaning of rational exponents follows from extending the properties of integer exponents to those values, allowing for a notation for radicals in terms of rational exponents.

Key Concepts

  • a1/n = ⁿ√a (the nth root of a)
  • am/n = (ⁿ√a)m = ⁿ√(am)
  • Properties of integer exponents extend: am · an = am+n, (am)n = amn
  • These properties motivate: (a1/n)n = a1 = a → a1/n must be the nth root

Example

Simplify 82/3: 82/3 = (81/3)² = (∛8)² = 2² = 4.

Or: 82/3 = (8²)1/3 = 641/3 = ∛64 = 4 ✓

N-RN.2

Rewriting Radicals and Rational Exponents

Rewrite expressions involving radicals and rational exponents using the properties of exponents.

Key Concepts

  • Convert between radical and rational exponent form freely
  • √a = a1/2; ∛a = a1/3; ⁴√a = a1/4
  • Simplify using exponent rules: combine like bases, distribute over products
  • Rationalize denominators when needed

Example

Simplify x3/4x1/4 = x3/4 − 1/4 = x2/4 = x1/2 = √x.

Simplify √x · ∛x = x1/2 · x1/3 = x3/6 + 2/6 = x5/6 = ⁶√x⁵.

N-CN.1

The Imaginary Unit i

Know there is a complex number i such that i² = −1, and every complex number has the form a + bi with a and b real.

Key Concepts

  • i = √(−1), defined so that i² = −1
  • Complex number: a + bi where a is the real part, b is the imaginary part
  • Powers of i cycle: i¹ = i, i² = −1, i³ = −i, i⁴ = 1, i⁵ = i, …
  • To find in: divide n by 4, use the remainder (0→1, 1→i, 2→−1, 3→−i)
  • Real numbers: b = 0. Pure imaginary: a = 0.

Example

Simplify i²³: 23 ÷ 4 = 5 remainder 3, so i²³ = i³ = −i.

Write √(−9) in complex form: √(−9) = √9 · √(−1) = 3i.

N-CN.2

Operations with Complex Numbers

Use the relation i² = −1 and the commutative, associative, and distributive properties to add, subtract, and multiply complex numbers.

Key Concepts

  • Add/Subtract: combine real parts and imaginary parts separately
  • Multiply: use FOIL, then replace i² with −1
  • Complex conjugate of a + bi is a − bi; product (a+bi)(a−bi) = a² + b² (real)
  • Divide: multiply numerator and denominator by conjugate of denominator

Example

(3 + 2i)(1 − 4i) = 3 − 12i + 2i − 8i² = 3 − 10i − 8(−1) = 3 + 8 − 10i = 11 − 10i.

Divide (2+i)/(1−i): multiply by (1+i)/(1+i) = (2+2i+i+i²)/(1+1) = (2+3i−1)/2 = (1+3i)/2.

N-CN.7

Quadratic Equations with Complex Solutions

Solve quadratic equations with real coefficients that have complex solutions.

Key Concepts

  • When the discriminant b² − 4ac < 0, the equation has two complex (non-real) solutions
  • Complex solutions always come in conjugate pairs: a + bi and a − bi
  • Use the quadratic formula: x = −b ± √(b²−4ac)2a, then write √(negative) using i

Example

Solve x² + 4x + 13 = 0: discriminant = 16 − 52 = −36. x = −4 ± √(−36)2 = −4 ± 6i2 = −2 ± 3i.

N-CN.8

Polynomial Identities Extended to Complex Numbers

Extend polynomial identities to the complex numbers. For example, rewrite x² + 4 as (x + 2i)(x − 2i).

Key Concepts

  • The factoring pattern a² − b² = (a+b)(a−b) extends when b is imaginary
  • Sum of squares: x² + a² = (x + ai)(x − ai)
  • This is only possible in the complex number system, not over the reals

Example

Factor x² + 9: = x² − (−9) = x² − (3i)² = (x + 3i)(x − 3i). Verify: (x+3i)(x−3i) = x² − 9i² = x² + 9 ✓.

N-CN.9

Fundamental Theorem of Algebra

Know the Fundamental Theorem of Algebra; show that it is true for quadratic polynomials.

Key Concepts

  • Fundamental Theorem: every non-constant polynomial with complex coefficients has at least one complex root
  • A degree-n polynomial has exactly n roots (counting multiplicity) in the complex numbers
  • Real-coefficient polynomials: complex roots come in conjugate pairs
  • Consequence: every polynomial factors completely over the complex numbers

Example

x⁴ − 1 = (x²−1)(x²+1) = (x−1)(x+1)(x−i)(x+i). Four roots: 1, −1, i, −i. The degree-4 polynomial has exactly 4 roots ✓.

A-SSE.1

Interpreting Expressions in Context

Interpret parts of an expression, such as terms, factors, and coefficients, in context. Interpret complicated expressions by viewing one or more of their parts as a single entity.

Key Concepts

  • In Algebra 2, expressions often involve polynomial, rational, logarithmic, and trigonometric forms
  • View complex sub-expressions as a unit to simplify interpretation
  • Connect algebraic form to real-world meaning (e.g., in finance, science, engineering)

Example

In the compound interest formula A = P(1 + r/n)nt: P = principal, r/n = rate per period, nt = total number of periods, (1 + r/n)nt = growth factor. Each part has physical meaning.

A-SSE.2

Structure of Advanced Expressions

Use the structure of an expression to identify ways to rewrite it.

Key Concepts

  • Recognize patterns: sum/difference of cubes, grouping, substitution
  • Sum of cubes: a³ + b³ = (a+b)(a² − ab + b²)
  • Difference of cubes: a³ − b³ = (a−b)(a² + ab + b²)
  • Substitution: let u = x² to factor x⁴ − 5x² + 6 as u² − 5u + 6 = (u−2)(u−3)

Example

Factor x⁶ − 8: = (x²)³ − 2³ = (x²−2)(x⁴ + 2x² + 4) using difference of cubes with a = x², b = 2.

A-SSE.4

Geometric Series Formula

Derive the formula for the sum of a finite geometric series (when the common ratio is not 1), and use the formula to solve problems.

Key Concepts

  • Geometric series: a + ar + ar² + … + arn−1
  • Sum: Sn = a · 1 − rn1 − r (for r ≠ 1)
  • Derivation: S − rS = a − arn, so S(1−r) = a(1−rn)
  • Infinite geometric series (|r| < 1): S = a1 − r

Example

Sum of 5 + 10 + 20 + 40 + 80 (a=5, r=2, n=5): S = 5 · 1 − 2⁵1 − 2 = 5 · 1 − 32−1 = 5 · 31 = 155.

A-APR.1

Closure of Polynomials (Advanced)

Understand that polynomials form a system closed under addition, subtraction, and multiplication. In Algebra 2 this extends to operations that arise from factoring higher-degree polynomials.

Key Concepts

  • Polynomial operations always yield polynomials (closure property)
  • Division of polynomials may yield a rational expression (not always a polynomial)
  • Long division and synthetic division for dividing polynomials

Example

Divide x³ − 2x² + 3x − 4 by (x − 1) using synthetic division: coefficients 1, −2, 3, −4; root 1. Result: x² − x + 2 remainder −2. So x³−2x²+3x−4 = (x−1)(x²−x+2) − 2.

A-APR.2

Remainder Theorem

Know and apply the Remainder Theorem: for a polynomial p(x) and a number a, the remainder on division by (x − a) is p(a), so p(a) = 0 if and only if (x − a) is a factor of p(x).

Key Concepts

  • Remainder Theorem: p(a) equals the remainder when p(x) is divided by (x − a)
  • Factor Theorem: (x − a) is a factor of p(x) if and only if p(a) = 0
  • Use to test potential factors without full division

Example

Is (x − 2) a factor of p(x) = x³ − 3x² + x + 2? p(2) = 8 − 12 + 2 + 2 = 0. Yes! (x−2) is a factor. Factor out: p(x) = (x−2)(x²−x−1).

A-APR.3

Zeros of Polynomials and Graphing

Identify zeros of polynomials when suitable factorizations are available, and use the zeros to construct a rough graph of the function defined by the polynomial.

Key Concepts

  • Zeros of p(x) = values where p(x) = 0 → x-intercepts of graph
  • Multiplicity: if (x−a)k is a factor, x = a is a zero of multiplicity k
  • Odd multiplicity: graph crosses x-axis at zero
  • Even multiplicity: graph touches (bounces at) x-axis at zero
  • End behavior determined by leading term

Example

p(x) = (x+1)²(x−3). Zeros: x=−1 (mult. 2, bounce) and x=3 (mult. 1, cross). Leading term: x³ → end behavior up right, down left.

A-APR.4

Polynomial Identities

Prove polynomial identities and use them to describe numerical relationships. For example, the polynomial identity (x² + y²)² = (x² − y²)² + (2xy)².

Key Concepts

  • A polynomial identity is an equation true for all values of the variables
  • Prove by expanding and simplifying both sides algebraically
  • (a + b)² = a² + 2ab + b² (perfect square)
  • (a + b)(a − b) = a² − b² (difference of squares)
  • (x² + y²)² = (x²−y²)² + (2xy)² → Pythagorean triple generator

Example

Prove (a+b)³ = a³ + 3a²b + 3ab² + b³: expand (a+b)(a+b)² = (a+b)(a²+2ab+b²) = a³ + 2a²b + ab² + a²b + 2ab² + b³ = a³ + 3a²b + 3ab² + b³ ✓.

A-APR.5 (CA)

Binomial Theorem

Know and apply the Binomial Theorem for the expansion of (x + y)ⁿ in powers of x and y for a positive integer n, where x and y are any numbers, with coefficients determined by Pascal's Triangle.

Key Concepts

  • (x + y)n = Σ C(n,k) · xn−k · yk for k = 0 to n
  • C(n,k) = n!k!(n−k)! (binomial coefficients)
  • Pascal's Triangle: each entry is sum of the two above it; row n gives coefficients of (x+y)n
  • The kth term (from k=0): C(n,k) · xn−k · yk

Example

Expand (x + 2)³: C(3,0)x³ + C(3,1)x²(2) + C(3,2)x(4) + C(3,3)(8) = x³ + 6x² + 12x + 8.

A-APR.6

Rewriting Rational Expressions

Rewrite simple rational expressions in different forms; write a(x)/b(x) in the form q(x) + r(x)/b(x), where a(x), b(x), q(x), and r(x) are polynomials with the degree of r(x) less than the degree of b(x).

Key Concepts

  • Polynomial long division: divide numerator by denominator
  • Result: quotient + (remainder/divisor)
  • Check: divisor × quotient + remainder = numerator
  • Used to find oblique asymptotes of rational functions

Example

Divide (x² + 3x + 1) ÷ (x + 2): x² + 3x + 1 = (x+2)(x+1) − 1. So result = x+1 + −1x+2.

A-APR.7

Rational Expressions as a System

Understand that rational expressions form a system analogous to the rational numbers, closed under addition, subtraction, multiplication, and division by a nonzero rational expression; add, subtract, multiply, and divide rational expressions.

Key Concepts

  • Add/Subtract: find LCD, convert to common denominator, combine numerators
  • Multiply: multiply numerators, multiply denominators, then simplify
  • Divide: multiply by the reciprocal of the divisor
  • Factor first to identify common factors to cancel

Example

2x+1 + 3x−2 = 2(x−2) + 3(x+1)(x+1)(x−2) = 2x−4+3x+3(x+1)(x−2) = 5x−1(x+1)(x−2).

A-CED.1

Creating Equations in One Variable (Advanced)

Create equations and inequalities in one variable and use them to solve problems. In Algebra 2, this includes polynomial, rational, radical, exponential, and logarithmic equations.

Key Concepts

  • Exponential equations: use logarithms to solve (e.g., 2x = 50 → x = log₂50)
  • Logarithmic equations: exponentiate both sides or use log properties
  • Rational equations: multiply through by LCD, check for extraneous solutions
  • Radical equations: isolate radical, square both sides, check for extraneous solutions

Example

A population doubles every 5 years. Starting at 1000, when does it reach 8000? 1000 · 2t/5 = 8000 → 2t/5 = 8 = 2³ → t/5 = 3 → t = 15 years.

A-CED.4

Rearranging Advanced Formulas

Rearrange formulas to highlight a quantity of interest, including formulas involving exponentials, logarithms, and radical expressions.

Key Concepts

  • Apply inverse operations at each step (same as solving equations)
  • To isolate an exponent: take logarithm of both sides
  • To isolate a logarithm: exponentiate both sides
  • To isolate under a radical: raise both sides to the appropriate power

Example

Solve A = P · ert for t: A/P = ert → ln(A/P) = rt → t = ln(A/P)r.

A-REI.2

Rational and Radical Equations

Solve simple rational and radical equations in one variable, and give examples showing how extraneous solutions may arise.

Key Concepts

  • Rational equations: multiply both sides by LCD; solve resulting polynomial; check solutions in original
  • Radical equations: isolate radical; raise both sides to power of radical; check solutions
  • Extraneous solutions: arise from squaring both sides or multiplying by expressions that could be 0; must check ALL solutions in the original equation

Example

Solve √(x + 5) = x − 1: square both sides: x + 5 = x² − 2x + 1 → x² − 3x − 4 = 0 → (x−4)(x+1) = 0. Check x=4: √9 = 3 ✓. Check x=−1: √4 = −2? No (extraneous). Solution: x = 4.

A-REI.4

Quadratic Equations — Advanced

Solve quadratic equations by inspection, taking square roots, completing the square, the quadratic formula, and factoring. Recognize when the quadratic formula gives complex solutions and write them in a + bi form.

Key Concepts

  • All methods from Algebra 1 apply; in Algebra 2, solutions may be complex
  • When discriminant < 0: solutions are complex conjugates a ± bi
  • Relate the discriminant to the number and type of solutions

Example

Solve 2x² − 3x + 5 = 0: discriminant = 9 − 40 = −31 < 0. x = 3 ± √(−31)4 = 3 ± i√314.

A-REI.6

Systems of Equations (Advanced)

Solve systems of linear equations exactly and approximately. In Algebra 2, extend to systems involving three variables.

Key Concepts

  • Three-variable systems: use elimination to reduce to two-variable, then solve
  • Matrix methods: augmented matrix with row operations (Gaussian elimination)
  • Solutions: unique point, no solution (inconsistent), infinitely many (dependent)

Example

System: x+y+z=6, 2x−y+z=3, x+2y−z=5. From eq1 and eq2 eliminate y: 3x+2z=9. From eq1 and eq3 eliminate z: 2x+3y=11. Continue to find x=1, y=2, z=3.

A-REI.7

Solving Nonlinear Systems

Solve a simple system consisting of a linear equation and a quadratic equation in two variables algebraically and graphically.

Key Concepts

  • Substitute the linear equation into the quadratic; solve the resulting quadratic
  • May have 0, 1, or 2 real solutions (tangent, intersecting, or non-intersecting)
  • Graphically: find intersection of the line and parabola

Example

System: y = x + 2 and y = x² − 4. Substitute: x + 2 = x² − 4 → x² − x − 6 = 0 → (x−3)(x+2) = 0. x = 3 → y = 5; x = −2 → y = 0. Two solutions: (3, 5) and (−2, 0).

A-REI.11

Intersections and Solutions

Explain why the x-coordinates of the points where the graphs of y = f(x) and y = g(x) intersect are the solutions of the equation f(x) = g(x); find the solutions approximately, using technology.

Key Concepts

  • In Algebra 2, f and g may be polynomial, exponential, logarithmic, or trig functions
  • Algebraic solutions may not be possible — numerical or graphical methods used
  • Each x-intersection is a solution to f(x) = g(x)

Example

Find where ex = x + 2: graph both functions; they intersect at approximately x ≈ 1.15 and x ≈ −1.84. No simple algebraic solution — use technology or Newton's method.

F-IF.4

Key Features of Advanced Functions

Interpret key features of graphs and tables for polynomial, rational, exponential, logarithmic, and trigonometric functions in terms of context.

Key Concepts

  • Rational functions: vertical asymptotes (where denominator = 0), horizontal asymptotes (end behavior), holes (cancelled factors)
  • Exponential: horizontal asymptote, growth/decay rate
  • Logarithmic: vertical asymptote, domain x > 0
  • Trig: period, amplitude, midline, phase shift

Example

f(x) = x + 1x − 3: vertical asymptote x = 3; horizontal asymptote y = 1 (degrees equal, ratio of leading coefficients); x-intercept at x = −1; y-intercept at y = −1/3.

F-IF.6

Rate of Change — Advanced Functions

Calculate and interpret the average rate of change of a function over a specified interval, including polynomial, exponential, and trigonometric functions.

Key Concepts

  • Same formula: f(b) − f(a)b − a
  • For exponential: rate of change is not constant — it grows proportionally to the function value
  • Interpret in context with appropriate units

Example

f(x) = 2x over [0, 3]: (f(3)−f(0))/(3−0) = (8−1)/3 = 7/3 ≈ 2.33. This is the average growth per unit — much less than the instantaneous rate at x=3.

F-IF.7

Graphing Advanced Functions

Graph polynomial functions, showing zeros and end behavior; graph rational functions, showing intercepts, asymptotes, and end behavior; graph exponential and logarithmic functions; graph trigonometric functions, showing period, midline, and amplitude.

Key Concepts

  • Polynomial: end behavior from leading term; zeros from factored form
  • Rational: vertical asymptotes from denominator zeros; horizontal from degree comparison
  • Exponential y = abx: y-intercept at (0,a); horizontal asymptote y=0
  • Logarithm y = logb(x): vertical asymptote x=0; passes through (1,0) and (b,1)
  • y = A·sin(Bx + C) + D: amplitude |A|, period 2π/B, phase shift −C/B, midline y=D

Example

y = 3·sin(2x) + 1: amplitude = 3, period = π, midline y = 1. Graph oscillates between y=−2 and y=4.

F-IF.8

Equivalent Forms — Advanced

Write a function in different but equivalent forms to reveal different properties. Use the properties of exponents to interpret expressions for exponential functions.

Key Concepts

  • f(x) = a · bx = a · ex·ln(b) (natural base equivalent)
  • Monthly compounding vs. continuous: A = P(1+r/12)12t vs. A = Pert
  • Factor polynomials into different forms to reveal zeros vs. vertex

Example

f(x) = 100 · (0.5)x/3. Rewrite: (0.5)x/3 = (0.51/3)x ≈ (0.794)x. This shows the per-period decay factor, revealing f decreases by about 20.6% per unit of x.

F-IF.9

Comparing Advanced Functions

Compare properties of two functions each represented in a different way (algebraically, graphically, numerically, or verbally), including polynomial, exponential, logarithmic, and trigonometric functions.

Key Concepts

  • Compare: domain, range, zeros, asymptotes, rate of change, periodicity
  • Translate between representations to extract comparable properties

Example

Function A: graph of exponential decay with horizontal asymptote y=0. Function B: f(x) = 5 · 0.8x. Compare y-intercepts: read from graph vs. f(0)=5. Compare decay rates: read steepness from graph vs. 20% per unit from formula.

F-BF.1

Writing Advanced Functions

Write a function that describes a relationship between two quantities, including combining function types using arithmetic operations and composition.

Key Concepts

  • Composition: f(g(x)) — apply g first, then f to the result
  • Domain of f(g(x)): x values where g(x) is defined AND in the domain of f
  • Combine functions: revenue = price × quantity; each may be a function of another variable

Example

f(x) = x² and g(x) = x + 3. f(g(x)) = f(x+3) = (x+3)² = x²+6x+9. g(f(x)) = g(x²) = x²+3. Note f(g(x)) ≠ g(f(x)) — composition is generally not commutative.

F-BF.2

Sequences and Series

Write arithmetic and geometric sequences recursively and explicitly; use them to model situations; translate between the two forms. Connect to function families.

Key Concepts

  • Arithmetic: an = a₁ + (n−1)d → linear function of n
  • Geometric: an = a₁ · rn−1 → exponential function of n
  • Partial sums of arithmetic: Sn = n2(a₁ + aₙ)
  • Partial sums of geometric: Sn = a₁ · 1 − rⁿ1 − r

Example

$1000 invested at 5% annual interest: a₁=1000, r=1.05. After 10 years: a₁₀ = 1000 · 1.05⁹ ≈ $1551. This is a geometric sequence.

F-BF.3

Transformations of Advanced Functions

Identify the effect on the graph of replacing f(x) by f(x)+k, kf(x), f(kx), and f(x+k). Extend to trigonometric, exponential, and logarithmic functions.

Key Concepts

  • Rules apply to ALL function families — linear, quadratic, trig, exponential, etc.
  • y = A·sin(B(x−C)) + D: amplitude A, period 2π/B, phase shift C, vertical shift D
  • Logarithm: y = log(x−h) + k shifts the vertical asymptote to x = h

Example

y = log₂(x): shift right 3, up 1 → y = log₂(x−3) + 1. Vertical asymptote moves from x=0 to x=3. Point (1,0) moves to (4,1).

F-BF.4

Inverse Functions — Advanced

Find the inverse of a function. Verify by composition that two functions are inverses of each other. Restrict domains to make non-invertible functions invertible. Read inverse values from a table or graph.

Key Concepts

  • f⁻¹ exists only if f is one-to-one (passes horizontal line test)
  • Exponential and logarithm are inverse functions: logb(bx) = x
  • Restrict domain of y = x² to [0,∞) so inverse y = √x is defined
  • Verify: f(f⁻¹(x)) = x AND f⁻¹(f(x)) = x

Example

f(x) = ex. Find inverse: y = ex → x = ey → y = ln(x). So f⁻¹(x) = ln(x). Verify: eln(x) = x and ln(ex) = x ✓.

F-LE.4

Logarithms as Solutions to Exponential Equations

For exponential models, express as a logarithm the solution to abct = d where a, c, and d are numbers and the base b is 2, 10, or e; evaluate the logarithm using technology.

Key Concepts

  • logb(x) is the inverse of bx: logb(bx) = x
  • Natural log: ln(x) = loge(x); Common log: log(x) = log₁₀(x)
  • Change of base: logb(x) = ln(x)/ln(b) = log(x)/log(b)
  • Log properties: log(ab) = log(a)+log(b); log(a/b) = log(a)−log(b); log(aⁿ) = n·log(a)
  • Solve abct = d: divide by a, take log of both sides, use power rule

Example

Solve 5 · 23t = 160: 23t = 32 = 2⁵ → 3t = 5 → t = 5/3. Or: 3t = log₂(32) = 5/ln(2) · ln(2) = 5. Same result.

F-TF.1

Radian Measure

Understand radian measure of an angle as the length of the arc on the unit circle subtended by the angle.

Key Concepts

  • 1 radian = angle that subtends an arc of length 1 on the unit circle (radius = 1)
  • Full circle: 2π radians = 360°; π rad = 180°
  • Convert degrees to radians: multiply by π/180
  • Convert radians to degrees: multiply by 180/π
  • Common: 30°=π/6, 45°=π/4, 60°=π/3, 90°=π/2, 180°=π

Example

Convert 150° to radians: 150 × π/180 = 5π/6. Convert 3π/4 to degrees: 3π/4 × 180/π = 135°.

F-TF.2

The Unit Circle

Explain how the unit circle in the coordinate plane enables the extension of trigonometric functions to all real numbers, interpreted as radian measures of angles traversed counterclockwise around the unit circle.

Key Concepts

  • Unit circle: circle of radius 1 centered at origin; (cos θ, sin θ) is the point at angle θ
  • cos θ = x-coordinate, sin θ = y-coordinate on the unit circle
  • Values extend beyond 0 to 2π by cycling — period of sin and cos is 2π
  • Quadrant signs: Q1 both +; Q2 sin +, cos −; Q3 both −; Q4 sin −, cos +
  • Key points: (0°/0, (1,0)), (90°/π/2, (0,1)), (180°/π, (−1,0)), (270°/3π/2, (0,−1))

Example

Find cos(2π/3): 2π/3 = 120° — in Q2. Reference angle 60°: cos(60°) = 1/2, negative in Q2. So cos(2π/3) = −1/2. sin(2π/3) = sin(60°) = √3/2 (positive in Q2).

F-TF.5

Modeling with Trigonometric Functions

Choose trigonometric functions to model periodic phenomena with specified amplitude, frequency, and midline.

Key Concepts

  • General form: y = A·sin(B(x − C)) + D or y = A·cos(B(x − C)) + D
  • Amplitude |A|: half the distance from min to max
  • Period 2π/B: horizontal length of one full cycle
  • Phase shift C: horizontal shift; positive = shift right
  • Vertical shift (midline) D: center line of oscillation

Example

Ocean tides: max 10 ft, min 2 ft, period 12 hours, starts at min at t=0. Amplitude = 4, midline = 6. Use cosine (starts at min with negative A): y = −4·cos(2π/12 · t) + 6 = −4·cos(πt/6) + 6.

F-TF.8

Pythagorean Identity

Prove the Pythagorean identity sin²(θ) + cos²(θ) = 1 and use it to find sin(θ), cos(θ), or tan(θ) given sin(θ) or cos(θ) and the quadrant of the angle.

Key Concepts

  • sin²θ + cos²θ = 1 (follows from the Pythagorean theorem on the unit circle)
  • Dividing by cos²θ: tan²θ + 1 = sec²θ
  • Dividing by sin²θ: 1 + cot²θ = csc²θ
  • Use quadrant information to determine the correct sign of the unknown ratio

Example

Given sin θ = 3/5 and θ is in Quadrant II. cos²θ = 1 − 9/25 = 16/25. cos θ = ±4/5. In Q2, cos is negative → cos θ = −4/5. tan θ = sin/cos = (3/5)/(−4/5) = −3/4.

S-ID.4

Normal Distribution

Use the mean and standard deviation of a data set to fit it to a normal distribution and to estimate population percentages. Recognize that there are data sets for which such a procedure is not appropriate.

Key Concepts

  • Normal distribution: bell-shaped, symmetric about the mean μ
  • Standard deviation σ measures spread
  • 68-95-99.7 Rule: 68% within 1σ, 95% within 2σ, 99.7% within 3σ of mean
  • Z-score: z = (x − μ)/σ — number of standard deviations from mean
  • Not all data is normal — skewed distributions require different methods

Example

Test scores: μ=75, σ=10. What percent score between 65 and 85? This is within 1σ of mean → approximately 68%. What percent score above 95? 95 is 2σ above mean → about 2.5%.

S-IC.1

Statistics as a Process

Understand statistics as a process for making inferences about population parameters based on a random sample from that population.

Key Concepts

  • Population: the entire group of interest
  • Parameter: a numerical summary of the population (e.g., population mean μ)
  • Sample: a subset of the population actually measured
  • Statistic: a numerical summary of the sample (e.g., sample mean x̄)
  • Random samples allow valid inferences about the population

Example

A school of 800 students wants to know average sleep. They randomly survey 80 students and find x̄ = 7.2 hours. This statistic estimates the population parameter μ (true average for all 800).

S-IC.2

Evaluating Statistical Models

Decide if a specified model is consistent with results from a given data-generating process, e.g., using simulation.

Key Concepts

  • If a result is very unlikely under the assumed model, question the model
  • Simulation: generate many samples from the model; compare observed result to simulated distribution
  • A result far in the tail of the simulated distribution suggests the model may be wrong

Example

A coin claimed to be fair is flipped 100 times and lands heads 70 times. Simulate 100 coin flips 1000 times. If getting ≥70 heads is very rare (<1% of simulations), the coin is likely not fair.

S-IC.3

Types of Statistical Studies

Recognize the purposes of and differences among sample surveys, experiments, and observational studies; explain how randomization relates to each.

Key Concepts

  • Survey: ask questions; estimates population characteristics; random sampling essential
  • Observational study: observe without intervening; can show association, not causation
  • Experiment: researcher assigns treatments; random assignment allows causal conclusions
  • Random assignment: controls for confounding variables in experiments

Example

To test a new drug: survey would just ask people if they feel better (bias); observational would compare who chose to take the drug vs. not (confounding); a randomized experiment (randomly assign drug vs. placebo) is the gold standard for causal inference.

S-IC.4

Sample Survey Data and Margin of Error

Use data from a sample survey to estimate a population mean or proportion; develop a margin of error through the use of simulation models for random sampling.

Key Concepts

  • Sample proportion p̂ estimates population proportion p
  • Margin of error (MOE): how far the estimate might be from the true value
  • Larger samples → smaller margin of error → more precise estimate
  • 95% confidence interval: p̂ ± MOE (contains true p about 95% of the time)
  • Approximate MOE for proportions: ≈ 1/√n

Example

Poll of 400 voters: 52% favor candidate A. MOE ≈ 1/√400 = 1/20 = 5%. 95% CI: 52% ± 5% = (47%, 57%). The true proportion is likely in this range.

S-IC.5

Randomized Experiment Data

Use data from a randomized experiment to compare two treatments; use simulations to decide if differences between parameters are significant.

Key Concepts

  • Observed difference between treatment and control could be due to random chance
  • Simulate: randomly re-assign subjects to groups many times; see how often you get a difference at least as large
  • If the observed difference is very rare in simulation → statistically significant → evidence of a real treatment effect

Example

Treatment group improves by 8 points on average vs. control. Simulate 1000 random reassignments; only 2% produce a difference ≥ 8. The result is statistically significant (p ≈ 0.02).

S-IC.6

Evaluating Statistical Reports

Evaluate reports based on data by looking at the study design, data collection method, and conclusions drawn.

Key Concepts

  • Check: Who was sampled? How? Was it random?
  • Check: Was it an experiment or observational study? (limits causal claims)
  • Watch for: biased sampling, leading questions, cherry-picked data, confounding
  • Check: Are conclusions supported by the data and design?

Example

Headline: "Students who eat breakfast score 20% higher on tests." This is observational — affluence may cause both breakfast-eating and test scores. The report should not claim breakfast causes better scores.

S-CP.1

Events as Subsets of a Sample Space

Describe events as subsets of a sample space using characteristics of the outcomes, or as unions, intersections, or complements of other events ("or," "and," "not").

Key Concepts

  • Sample space S: all possible outcomes
  • Event A: a subset of S
  • Union A ∪ B: A OR B (either or both happen)
  • Intersection A ∩ B: A AND B (both happen)
  • Complement Aᶜ: NOT A (A does not happen)

Example

Roll a die. S = {1,2,3,4,5,6}. A = even = {2,4,6}. B = >3 = {4,5,6}. A∩B = {4,6}; A∪B = {2,4,5,6}; Aᶜ = {1,3,5}.

S-CP.2

Independent Events

Understand that two events A and B are independent if the probability of A and B occurring together is the product of their probabilities, and use this characterization to determine if they are independent.

Key Concepts

  • A and B are independent if P(A ∩ B) = P(A) · P(B)
  • Equivalently: P(A|B) = P(A) — knowing B doesn't affect probability of A
  • Independent ≠ mutually exclusive: if A and B are mutually exclusive (and both possible), they are NOT independent

Example

Flip a coin and roll a die. P(H) = 1/2, P(6) = 1/6. P(H and 6) = 1/12 = P(H)·P(6) ✓ → independent. Drawing cards without replacement: not independent (probabilities change).

S-CP.3

Conditional Probability

Understand the conditional probability of A given B as P(A and B)/P(B), and interpret independence of A and B as saying that the conditional probability of A given B is the same as the probability of A, and the conditional probability of B given A is the same as the probability of B.

Key Concepts

  • P(A|B) = P(A ∩ B)P(B) ("probability of A given B has occurred")
  • Restricts the sample space to only outcomes in B
  • Independence check: A ⊥ B iff P(A|B) = P(A)

Example

Draw a card. P(King | Face card): P(King and Face) = 4/52, P(Face) = 12/52. P(King|Face) = (4/52)/(12/52) = 4/12 = 1/3. (Makes sense: 4 kings out of 12 face cards.)

S-CP.4

Two-Way Tables and Probability

Construct and interpret two-way frequency tables of data when two categories are associated with each object being classified. Use the two-way table as a sample space to decide if events are independent and to approximate conditional probabilities.

Key Concepts

  • Read P(A), P(B), P(A ∩ B) from cell counts and totals
  • Test independence: does P(A ∩ B) ≈ P(A) · P(B)?
  • Conditional probability: row or column relative frequencies

Example

100 students: 40 athletes (30 pass, 10 fail), 60 non-athletes (35 pass, 25 fail). P(Pass|Athlete) = 30/40 = 75%. P(Pass|Non-athlete) = 35/60 ≈ 58%. Different rates suggest association between athletics and passing.

S-CP.5

Conditional Probability in Everyday Language

Recognize and explain the concepts of conditional probability and independence in everyday language and everyday situations.

Key Concepts

  • Real-world conditional probability: "given that it is raining, what is the chance of traffic?"
  • Independence in context: "does knowing someone's height tell us their test score?" (likely no → independent)
  • Translate plain English to probability notation

Example

"What is the probability that a randomly selected employee is female, given that they work in engineering?" This is P(Female | Engineering) — find it by dividing female engineers by all engineers.

S-CP.6

Finding Conditional Probability

Find the conditional probability P(A|B) as the fraction of B's outcomes that also belong to A, and interpret the answer in terms of the model.

Key Concepts

  • P(A|B) = (# outcomes in A and B) ÷ (# outcomes in B)
  • Equivalently: P(A|B) = P(A ∩ B) / P(B)
  • Use tables or lists to count outcomes directly

Example

Bag has 5 red, 3 blue, 2 green. One is drawn and is not green. P(Red | not green) = 5/(5+3) = 5/8. The sample space is restricted to the 8 non-green marbles.

S-CP.7

Addition Rule for Probability

Apply the Addition Rule, P(A or B) = P(A) + P(B) − P(A and B), and interpret the answer in terms of the model.

Key Concepts

  • P(A ∪ B) = P(A) + P(B) − P(A ∩ B)
  • Subtract P(A ∩ B) to avoid double-counting outcomes in both A and B
  • Mutually exclusive events: P(A ∩ B) = 0, so P(A ∪ B) = P(A) + P(B)

Example

P(draw a heart) = 13/52; P(draw a face card) = 12/52; P(heart and face) = 3/52 (J♥, Q♥, K♥). P(heart or face) = 13/52 + 12/52 − 3/52 = 22/52 = 11/26.

S-MD.6

Fair Decisions with Probability

Use probabilities to make fair decisions (e.g., drawing by lots, using a random number generator).

Key Concepts

  • A decision is fair if each outcome has the appropriate probability
  • Random number generators, coin flips, and random draws can implement fair decisions
  • Must verify that the method actually produces the desired probabilities

Example

5 students must each give a 5-minute presentation. To assign a fair random order, assign each a number 1–5, then use a random number generator (without replacement). Each student has equal probability of any position.

S-MD.7

Analyzing Decisions Using Probability

Analyze decisions and strategies using probability concepts (e.g., product testing, medical testing).

Key Concepts

  • Medical testing: sensitivity (P(positive | disease)), specificity (P(negative | no disease))
  • False positive: test positive when disease absent; false negative: test negative when disease present
  • Expected value: used to analyze decisions — pick option with highest expected payoff
  • E(X) = Σ [x · P(X = x)]

Example

Game: win $10 with probability 0.3, lose $4 with probability 0.7. E = 10(0.3) + (−4)(0.7) = 3 − 2.8 = $0.20. Positive expected value → favorable game on average.
